# Logs Directory

This directory contains application logs generated by the GoldMiner ETL pipeline.

## Log Files

- **goldminer.log**: Main application log file
  - Contains detailed pipeline execution logs
  - Includes INFO, WARNING, and ERROR messages
  - Configured in `config.yaml`

## Log Configuration

The logging behavior can be customized in `config.yaml`:

```yaml
logging:
  level: INFO          # DEBUG, INFO, WARNING, ERROR, CRITICAL
  format: "%(asctime)s - %(name)s - %(levelname)s - %(message)s"
  file: logs/goldminer.log
```

## Log Levels

- **DEBUG**: Detailed diagnostic information
- **INFO**: General informational messages (default)
- **WARNING**: Warning messages for potentially problematic situations
- **ERROR**: Error messages for serious problems
- **CRITICAL**: Critical messages for very serious errors

## Viewing Logs

```bash
# View recent logs
tail -f logs/goldminer.log

# View last 100 lines
tail -n 100 logs/goldminer.log

# Search for errors
grep ERROR logs/goldminer.log
```

## Notes

- Log files in this directory are gitignored by default
- The directory is created automatically if it doesn't exist
- Logs are appended to the file, not overwritten
- Consider rotating logs periodically for large-scale operations
